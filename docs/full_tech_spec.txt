Общая информация о проекте
1.1 Цель проекта
Разработка современной premium e-commerce платформы для продажи сладких подарков на
европейском рынке. Платформа обслуживает розничных покупателей (B2C) и корпоративных
клиентов (B2B), обеспечивая уникальный пользовательский опыт в категории «домашний
премиум» — тёплые европейские традиции, качественная кондитерская продукция,
персонализированные подарочные решения.
1.2 Целевая аудитория
B2C (розница):
◆ Частные покупатели в странах EU: Германия, Франция, Испания, Италия, другие
◆ Возраст 25–55 лет, средний и высокий доход
◆ Ищут качественные подарки на праздники, дни рождения, корпоративные поводы
◆ Ценят удобство онлайн-шопинга и возможность персонализации
B2B (корпоративные):
◆ Компании, закупающие подарки для сотрудников, клиентов, партнёров
◆ Розничные магазины и специализированные бутики
◆ Event-агентства и организаторы корпоративных мероприятий
◆ Рестораны и отели класса «люкс»

Глоссарий
Определения терминов, используемых в настоящем техническом задании.
Термин
 Определение
 API
Интерфейс программирования приложений — набор протоколов, позволяющих
программным компонентам взаимодействовать.
App Router
Система маршрутизации Next.js 14 на основе файловой структуры с поддержкой React
Server Components и Layouts.
B2B
Business-to-Business — модель продажи товаров и услуг между юридическими лицами.
B2C
Business-to-Consumer — модель прямых продаж от компании конечному потребителю.
CDN
Content Delivery Network — сеть распределённых серверов для быстрой доставки контента
из ближайшего узла.
CI/CD
Continuous Integration / Continuous Deployment — автоматическая проверка и развёртывание
кода при каждом изменении.
CLS
Cumulative Layout Shift — метрика Core Web Vitals, визуальная стабильность страницы.
Норма: < 0.1.
CMS
Content Management System — система управления контентом без написания кода.
Core Web Vitals
Ключевые метрики Google для SEO: LCP, CLS, INP. Влияют на ранжирование сайта.
CSRF
Cross-Site Request Forgery — атака на веб-приложение. Защита — CSRF-токены.
DPA
Data Processing Agreement — договор об обработке данных, обязательный по нормам GDPR.
E2E Testing
End-to-End тестирование — проверка полного пользовательского сценария от начала до
конца.
Edge Runtime
Выполнение кода на серверах CDN близко к пользователю для минимальной задержки.
GDPR
General Data Protection Regulation (ЕС) 2016/679 — регламент о защите персональных
данных граждан ЕС.
GraphQL
Язык запросов к API, позволяющий клиенту запрашивать только нужные данные.
GROQ
Graph-Relational Object Queries — язык запросов Sanity.io для выборки данных из CMS.
Headless CMS
CMS с отдельным бэкендом (API) и фронтендом — контент доступен для любой платформы.
hreflang
HTML-атрибут, указывающий поисковикам языковую/региональную версию страницы для
SEO.
i18n
Internationalization — проектирование приложения с возможностью адаптации к разным
языкам без изменения кода.
INP
Interaction to Next Paint — метрика отзывчивости интерфейса. Норма: < 200мс.
ISR
Incremental Static Regeneration — обновление статических страниц Next.js в фоне без полной
пересборки.
JWT
JSON Web Token (RFC 7519) — стандарт токенов аутентификации с цифровой подписью.
LCP
Largest Contentful Paint — время появления наибольшего видимого элемента. Норма: < 2.5
сек.
Lighthouse
Инструмент Google для аудита производительности, SEO, доступности. Шкала 0–100.
MOQ
Minimum Order Quantity — минимальный объём заказа для B2B клиентов.
NestJS
TypeScript фреймворк для Node.js с модульной архитектурой для серверных приложений.
Next.js
React фреймворк с SSR, SSG, ISR и API Routes. Разрабатывается Vercel.
OAuth 2.0
Открытый стандарт авторизации («Войти через Google»). Позволяет делегировать доступ
без передачи пароля.

Webhook

XSS
Cross-Site Scripting — инъекция вредоносного скрипта в страницу. Защита: CSP и
